## DIY Voreinstellungen

Bewährte Hardwarekombinationen mit vorgefertigten Konfigurationen.

### Unterstützte Hardware

#### Basisgerät

- Heltec Mesh Node T114
- NRF52 Pro-micro DIY

#### Anzeigemodule

- WeAct Studio 1,54 Zoll B&W
- WeAct Studio 2,13 Zoll B&W (2025 Revision)
- WeAct Studio 2,9 Zoll B&W
- WeAct Studio 4,2 Zoll B&W

### Firmware

Obwohl Konfigurationen für diese Geräte bereitgestellt werden, müssen Sie dennoch die Meshtastic Firmware erstellen.

#### Erstellen mit Visual Studio Code

Zuerst die [Build-Umgebung einrichten](/docs/development/firmware/build/#build), dann:

1. Öffnen Sie den Bereich „PlatformIO Projektaufgaben“.

2. Wählen Sie die Umgebung aus, die zu Ihrer Knotenhardware passt
   - `heltec-mesh-node-t114-inkhud`
   - `nrf52_promicro_diy-inkhud`

3. Führen Sie die Build Aufgabe aus, die zu Ihrem Anzeigemodell passt.
   ![Benutzerdefinierte PlatformIO Build Aufgaben für DIY Voreinstellungen](/img/software/inkhud/diy-presets.webp)

4. Sobald die Kompilierung abgeschlossen ist, finden Sie die generierte .uf2-Datei unter `./pio/build/<environment>/firmware.uf2`
   ![uf2-Ausgabedatei des Firmware-Builds](/img/software/inkhud/firmware-build-output.webp)

#### Kompilieren mit der Platformio Kommandozeile

Führen Sie `pio run -e <environment> -t <build-task>` aus, wobei:

`<environment>`

- `heltec-mesh-node-t114-inkhud`
- `nrf52_promicro_diy-inkhud`

`<build task>`

- `build_weact_154`
- `build_weact_213`
- `build_weact_290`
- `build_weact_420`

Suchen Sie dann die generierte .uf2-Datei unter `./pio/build/<environment>/firmware.uf2`

### Installation

Installieren Sie die generierte .uf2-Datei mithilfe der [Ziehen und Ablegen Methode](/docs/getting-started/flashing-firmware/nrf52/drag-n-drop/#nrf52) auf dem Gerät.

## Andere Hardware

Die Verwendung von InkHUD auf Hardware über die standardmäßigen DIY Voreinstellungen hinaus erfordert eine manuelle Anpassung der Firmware. 
Hier kann keine konkrete Hilfestellung gegeben werden, sondern nur ein allgemeiner Überblick.

**Nach einem Anzeigetreiber suchen**

InkHUD basiert auf benutzerdefinierten Anzeigetreibern. 
Diese befinden sich in `/src/graphics/niche/Drivers/EInk/`.
Wählen Sie eine Anzeige, die bereits unterstützt wird. Andernfalls müssen Sie den Treiber selbst schreiben, was einige Fachkenntnisse erfordert.

**Klonen und Ändern einer vorhandenen InkHUD Variante**

Hier gelten die allgemeinen Anweisungen zum [Bauen mit benutzerdefinierter Hardware](/docs/development/firmware/build/#build-with-custom-hardware).
Zusätzlich zur Änderung der Datei `variant.h` sollten Sie auch [die Datei `nicheGraphics.h` ändern](https://github.com/meshtastic/firmware/tree/master/src/graphics/niche/InkHUD/docs#nichegraphicsh), die für die variantenspezifische Konfiguration von InkHUD verantwortlich ist.
