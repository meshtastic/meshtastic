Das Meshtastic Ökosystem ist gut erweiterbar, und es wurden eine Reihe von Gemeinschaftsprojekten gegründet, um die verschiedenen Bedürfnisse der Nutzer zu erfüllen. Wenn Sie Ihre eine eigene Anwendung oder Ihr eigenes Modul erstellen möchten, lesen Sie bitte die Informationen im Abschnitt für Entwickler und stellen Sie uns Ihr Projekt im Forum vor.

Aktuelle Gemeinschaftsprojekte:

- [Meshtastic Web API](/docs/community/software/meshtastic-web-api) - Meshtastic Web API bietet eine REST Schnittstelle zur Interaktion mit einem Meshtastic Knoten über eine serielle Verbindung.
- [Mesh Metriken](/docs/community/software/community-mesh-metrics) - Meshtastic Metriken benutzen das Meshtastic Kommandozeileninterface (CLI) zum Aufnehmen durch node_exporter in Kombination mit Prometheus oder die Victoria Metrics Datenbank mit zeitlichem Verlauf in einer Grafana Oberfläche, inklusive Regeln zur Alarmierung.
- [Node-RED Messages Node](/docs/community/software/community-node-red-messages) - Node-RED Applikation zum Senden und Empfangen von Nachrichten über ein Gerät das via HTTP verbunden ist.
- [Telemetry Harbor - Grafana](/docs/community/software/telemetry-harbor) - Eine Cloud-basierte Lösung zum Erfassen, Speichern und Visualisieren von Meshtastic Telemetriedaten mit Grafana.
- [Connect](https://github.com/pdxlocations/connect) - Eine eigenständige Python Software für die Kommunikation mit Meshtastic Geräten über MQTT.
- [Contact](https://github.com/pdxlocations/contact) - Eine terminal-basierte Python-Chat und Konfigurationssoftware für Meshtastic Geräte.

Unterstützung für diese Projekte sollte bei ihren jeweiligen Autoren erfragt werden.
