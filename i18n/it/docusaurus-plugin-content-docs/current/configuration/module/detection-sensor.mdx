Il modulo Sensore di Rilevamento consente di configurare un pin GPIO da monitorare per uno stato alto/basso specificato e di inviare avvisi di testo su portnum del Sensore di Rilevamento, quando viene rilevato un evento. Ciò è particolarmente utile per i sensori di rilevamento del movimento, gli interruttori magnetici (reed) e altri sistemi a stato aperto/chiuso in cui sono desiderate le notifiche sulla mesh. Le opzioni di configurazione sono: Abilitato, Intervallo Minimo di Trasmissione, Intervallo Trasmissione Stato, Invia Campanella, Nome, Pin di Monitoraggio, Rilevamento Attivato su Alto, e Usa Pull-up.

Per utilizzare questo modulo, assicurati che i tuoi dispositivi abbiano la versione del firmware 2.2.2 o superiore.

## Valori Di Configurazione Del Modulo Del Sensore Di Rilevamento

### Abilitato

Se il modulo è abilitato.

### Intervallo Minimo Di Trasmissione

L'intervallo in secondi di quanto spesso possiamo inviare un messaggio alla mesh quando viene rilevato un cambiamento di stato.

### Intervallo Di Trasmissione

L'intervallo in secondi che determina quanto spesso inviare un messaggio alla mesh con lo stato attuale indipendentemente dai cambiamenti. Quando impostato a 0, verranno trasmessi solo i cambi di stato. Funziona come una sorta di heartbeat di stato per tranquillità.

### Invia Campanella

Invia campanella ASCII con messaggio di avviso. Utile per l'attivazione di notifiche esterne sul nome della campanella.

### Nome Descrittivo

Usato per formattare il messaggio inviato alla mesh. Esempio: Un nome "Motion" porterebbe a un messaggio "Motion detected". Lunghezza massima di 20 caratteri.

### Pin Monitor

Il pin GPIO da monitorare per i cambiamenti di stato.

### Rilevamento Trigger su HIGH

Determina se il rilevamento dello stato del pin GPIO si attiva su HIGH (1) o su LOW (0).

### Usa Pull-up

Indica se usare o meno la modalità INPUT_PULLUP per il pin GPIO. Applicabile solo se la scheda utilizza resistenze pull-up sul pin.

#### CLI

:::info

Tutte le opzioni di configurazione del modulo Paxcounter sono disponibili nella versione di python CLI 2.2.2 e successive.

:::

I comandi di esempio sono di seguito:

```shell title="Enable/Disable the Detection Sensor Module"
meshtastic --set detection_sensor.enabled true
meshtastic --set detection_sensor.enabled false
```

```shell title="Set the Minimum Broadcast Interval to 90 seconds"
meshtastic --set detection_sensor.minimum_broadcast_secs 90
```

```shell title="Set the State Broadcast Interval to 5 minutes"
meshtastic --set detection_sensor.state_broadcast_secs 300
```

```shell title="Enable/Disable Send Bell"
meshtastic --set detection_sensor.send_bell true
meshtastic --set detection_sensor.send_bell false
```

```shell title="Set the friendly name to 'motion'"
meshtastic --set detection_sensor.name "motion"
```

```shell title="Set the Monitor Pin to 7"
meshtastic --set detection_sensor.monitor_pin 7
```

```shell title="Enable Notifications when the Monitor Pin goes HIGH"
meshtastic --set detection_sensor.detection_triggered_high true
```

```shell title="Enable Notifications when the Monitor Pin goes LOW"
meshtastic --set detection_sensor.detection_triggered_high false
```

```shell title="Enable the use INPUT_PULLUP mode"
meshtastic --set detection_sensor.use_pullup true
```

```shell title="Get the Detection Sensor Module Configuration"
meshtastic --get detection_sensor
```
