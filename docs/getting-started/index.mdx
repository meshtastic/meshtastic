## How Meshtastic Works

Meshtastic lets you send messages, share GPS locations, and more — without needing cell service or internet. Each device uses a LoRa radio to communicate over long distances, forming a mesh network where messages hop between devices to extend their range. You control your device from a phone or computer using Bluetooth, WiFi, or USB.

<MeshTopology />

## What You'll Need

To get started with Meshtastic, you need:

1. **A Meshtastic-compatible radio** — See the hardware section below to identify your device.
2. **A USB data cable** — Many USB cables only charge; yours must also transfer data.
3. **A phone or computer** — To configure your device and send messages.

:::danger Antenna Required!

**Never power on your radio without an antenna attached.** Operating without an antenna can permanently damage the radio chip.

:::

## Step 1: Identify Your Hardware

Before setting up your device, you need to know which chip it uses. This determines how you'll install firmware and which drivers (if any) you need. Most Meshtastic devices use one of three chip families:

| Chip Family | Key Features | Firmware Method |
|---|---|---|
| **ESP32** | WiFi + Bluetooth, web interface support | [Web Flasher](https://flasher.meshtastic.org) (easiest) |
| **nRF52** | Very power-efficient, Bluetooth only | Drag & drop file copy |
| **RP2040/RP2350** | Affordable, great for DIY | Drag & drop file copy |

## Step 2: Check Your USB Cable

Not all USB cables are the same — many only charge and cannot transfer data. Before you plug in your device, verify your cable:

1. Connect the cable to another device (like a phone).
2. Try copying a file to or from that device.
3. If the file transfer works, your cable supports data transfer. ✅

If file transfers don't work, you'll need a different cable.

## Step 3: Install Serial Drivers (ESP32 Only)

:::tip nRF52 and RP2040/RP2350 users can skip this step

nRF52, RP2040, and RP2350 devices use a UF2 bootloader that makes them appear as USB flash drives. No special drivers are needed — you can go straight to [Step 4: Flash Firmware](#step-4-flash-firmware).

:::
