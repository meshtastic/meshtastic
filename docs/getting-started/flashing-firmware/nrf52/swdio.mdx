Some Meshtastic devices (e.g., the Seeed WM1110 Dev Kit) can't be flashed via USB and require a direct debug connection using SWDIO. While dedicated tools like the [RAKDAP1](https://msh.to/rakdap1/), [Segger J-Link](https://msh.to/segger-jlink), or [ST-Link](https://msh.to/st-link) work well, a Raspberry Pi can also serve as a SWDIO flash tool.

## Step 1: Wire the Connection

Connect two wires between your Raspberry Pi and the SWDIO ports on your nRF52-based device:

| Raspberry Pi | Meshtastic Device |
|---|---|
| GPIO 11 | CLK (Clock) |
| GPIO 8 | DIO (Data) |

:::caution
Double-check the GPIO pin numbers — incorrect wiring will prevent the connection from working.
:::

Power on both the Raspberry Pi (via USB) and the Meshtastic device.

## Step 2: Install OpenOCD

OpenOCD manages the SWDIO connection. Install it on your Raspberry Pi:

```shell
sudo apt update
sudo apt install libtool autoconf automake texinfo telnet gdb-multiarch git
git clone git://repo.or.cz/openocd.git
cd openocd/
./bootstrap
./configure --enable-bcm2835gpio
make -j4
sudo make install
```

## Step 3: Test the Connection

Before flashing, verify that OpenOCD can communicate with the device:

```shell
openocd -f ./openocd/tcl/interface/raspberrypi-native.cfg -c "transport select swd" -f openocd/tcl/target/nrf52.cfg -s tcl
```

You should see output like:

```
Open On-Chip Debugger 0.12.0+dev-00633-gad87fbd1c
Licensed under GNU GPL v2
...
Info : SWD DPIDR 0x2ba01477
Info : [nrf52.cpu] Cortex-M4 r0p1 processor detected
Info : [nrf52.cpu] target has 6 breakpoints, 4 watchpoints
Info : Listening on port 4444 for telnet connections
```

If you see `Error connecting DP: cannot read IDR`, the wires are likely connected to the wrong pins.

## Step 4: Get the Firmware

Check the instructions for your [hardware](/docs/hardware/devices) to find the correct firmware file. For most devices following this guide, you need a `.hex` file. Copy it to your Raspberry Pi.

## Step 5: Flash the Firmware

With OpenOCD still running, open a new terminal and connect via telnet:

```shell
telnet 127.0.0.1 4444
```

Then run these commands in the telnet session:

```
> init
> reset init
> halt
> nrf5 mass_erase
> program firmware.hex verify
> reset
> exit 0
```

After flashing, unplug and re-plug the power to the Meshtastic device.

## Done!

Your device should now be running Meshtastic firmware. Connect to it via Bluetooth using the [Meshtastic app](/docs/getting-started/initial-config) to complete setup.
