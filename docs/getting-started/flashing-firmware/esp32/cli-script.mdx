This guide walks you through flashing ESP32 firmware using command-line tools. Most users should use the [Web Flasher](/docs/getting-started/flashing-firmware/esp32/web-flasher) instead — it's faster and easier.

:::danger Antenna Required!

**Never power on your radio without an antenna attached.** This can permanently damage the radio chip.

:::

## Step 1: Verify Device Connection

Connect your device to your computer with a USB data cable, then confirm it's detected:

```shell title="Command"
esptool chip-id
```

```shell title="Expected Output"
esptool.py v4.x
Found 2 serial ports
Serial port /dev/ttyUSB0
Connecting....
Detecting chip type... ESP32
Chip is ESP32D0WDQ6 (revision 1)
Features: WiFi, BT, Dual Core, 240MHz, VRef calibration in efuse, Coding Scheme None
MAC: 24:6f:28:b5:36:71
```

:::tip esptool not found?

On Linux and macOS, you may need to use `esptool.py` instead of `esptool`.

:::

If the command fails, check that:
- You're using a data cable (not charge-only)
- [Serial drivers](/docs/getting-started/serial-drivers/esp32) are installed
- You've rebooted after installing drivers

## Step 2: Download Firmware

1. Go to the [Meshtastic Downloads](/downloads) page.
2. Choose **Download Stable** (recommended) or **Download Alpha** (experimental).
3. On the GitHub releases page, expand the **Assets** section.
4. Download the firmware zip file that matches your device architecture (e.g., `firmware-esp32s3` for ESP32-S3 devices like the Heltec V3).
5. Unzip the downloaded file.

## Step 3: Flash the Firmware

Open a terminal and navigate to the unzipped firmware folder:

```shell title="Example"
cd ~/Downloads/firmware-esp32s3-X.X.X.xxxxxxx/
```

:::note
Replace `esp32s3` and the version number with those matching your downloaded file.
:::

:::caution Flash the correct firmware for your board

Using the wrong firmware file can leave your device unresponsive with a blank screen. For example:
- The **T-Beam** requires `firmware-tbeam`, **not** `firmware-tlora` (that's a different board).
- The older [T-Beam 0.7](/docs/hardware/devices/lilygo/tbeam/?t-beam=0.7) requires `firmware-tbeam0.7`, which is different from other T-Beam firmware.

Check the [hardware section](/docs/hardware/devices/) to confirm the correct firmware for your device.

:::
